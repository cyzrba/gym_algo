# Task Status Field & State Machine Plan

## Overview
Add a `status` field to the Task model with state machine logic for status transitions.

## Files to Modify

### 1. `src/app/schemas/common.py` — 添加 TaskStatus 常量
- 添加 `TaskStatus` 类（使用 `str, Enum` 模式，方便 JSON 序列化）
- 状态值: `PENDING`, `PROCESSING`, `SUCCESS`, `FAIL`, `CANCEL`

### 2. `src/app/model/task.py` — Task 模型加 status 字段
- 添加 `status: str` 字段，默认值 `TaskStatus.PENDING`

### 3. `src/app/schemas/task.py` — Schema 加 status
- `TaskResponse` 加 `status: str` 字段
- `TaskCreate` / `TaskUpdate` 不加（status 由服务端控制）

### 4. `src/app/services/task.py` — 状态机逻辑
- `create_task`: 默认 status = PENDING
- 新增 `transition_task_status(session, task_id, new_status)` 函数
- 状态机规则:
  - PENDING → PROCESSING, CANCEL
  - PROCESSING → SUCCESS, FAIL, CANCEL
  - SUCCESS / FAIL / CANCEL → 终态，不可变更
- 非法转换抛 `ValueError`

### 5. `src/app/api/task.py` — API 更新
- 响应中包含 `status`
- 新增 `PATCH /{task_id}/status` 端点，接收 `{"status": "processing"}` 调用 `transition_task_status`

## Status Flow
```
pending → processing → success
                   ↘ fail
                   ↘ cancel
pending → cancel
```
