# 智能健身房人体围度测量系统 - 后端逻辑 PRD

## 1. 项目概述
本系统旨在通过 3D 视觉技术，自动化测量用户的**臂围、腰围、腿围**（及肩宽相关数据），消除传统皮尺测量的人为误差，为健身用户提供数字化的身体数据追踪。

## 3. 功能需求

### 3.1 数据输入与存储结构
后端需监听或接收前端上传的原始原始采集数据。
* **输入路径**：`/data/img/`
* **输入文件类型**：
    * `*.raw`：RGB 彩色图（用于视觉对齐或人体分割辅助）。
    * `*.raw`：Depth 深度图（包含空间深度信息）。
    * `*.ply`：Point Cloud 点云图（三维重建后的核心数据）。

### 3.2 核心处理模块
系统采用模块化设计，通过 `pointcloud.py` 提供基础计算支持，三个专项脚本实现具体部位测量。

| 模块名称 | 对应文件 | 功能描述 |
| :--- | :--- | :--- |
| **基础算法库** | `pointcloud.py` | 提供点云预处理（滤波、去噪、旋转平齐）、切片提取算法、多边形拟合及周长计算函数。 |
| **臂围处理** | `arm_pointcloud.py` | 定位大臂/小臂坐标，提取垂直于骨骼方向的点云切片，计算二头肌/三头肌处围度。 |
| **腿围处理** | `leg_pointcloud.py` | 定位大腿根部与小腿腹位置，计算最大腿围及膝盖上方围度。 |
| **躯干与肩部** | `shoulder_pointcloud.py` | 识别腰部最细处（腰围）及肩膀峰值宽度。 |

### 3.3 数据处理流水线 (Pipeline)
1.  **初始化**：调用 `pointcloud.py` 对输入的 `.ply` 文件进行归一化处理。
2.  **人体分割**：基于深度图和点云，识别并拆分人体组件（躯干、四肢）。
3.  **切片提取**：根据人体比例，在对应高度进行横截面切片。
4.  **周长拟合**：
    * 使用 Alpha Shape 或凸包算法提取边缘点。
    * 通过 B-Spline 或 Ellipse Fitting（椭圆拟合）计算闭合曲线周长 $C$。
5.  **结果输出**：将处理后的结果存入指定目录。

## 4. 输出与存储需求
* **输出路径**：`/data/result/`
* **生成内容**：
    1.  **JSON 报告**：记录臂围、腰围、腿围的具体数值（单位：mm）。
    2.  **可视化图示**：在 `result` 文件夹下生成带切片标注的 `.jpg` 预览图，方便用户直观核对测量位置。
    3.  **处理日志**：记录算法耗时与置信度评分。

---

## 5. 算法逻辑参考 (pointcloud.py)
基础库需实现的核心逻辑如下：

* **坐标系转换**：将传感器坐标系转换为人体中心坐标系。
* **周长计算公式**：
    对于非规则闭合曲线，采用离散积分方式：
    $$L \approx \sum_{i=1}^{n} \sqrt{(x_{i+1}-x_i)^2 + (y_{i+1}-y_i)^2}$$
    或者使用平滑后的参数方程进行积分。

## 6. 技术难点与注意事项
* **姿态矫正**：用户站姿如果不标准（如轻微侧身），`pointcloud.py` 必须具备自动旋转校正功能。
* **遮挡处理**：腋下和裆部是点云缺失的高发区，算法需要具备补全或鲁棒的曲线拟合能力。
* **并发处理**：健身房场景下，后端需支持多请求排队队列，避免内存溢出（点云数据通常较大）。
